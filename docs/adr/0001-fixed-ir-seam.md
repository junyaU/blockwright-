# IR seam `build(ir, origin)` を唯一の固定境界にする

AIの「賢さ」と決定論コードの「正確さ」を `build(ir, origin): BuildResult` 一本の境界で分け、その**署名と契約を不変**とする。表現力は `build.ts` の `switch` に型分岐を足して育て、下流（座標計算・コマンド生成・送信・Undo）は再実装しない。retrofit が最も高くつく継ぎ目はここ一点であり、ここさえ固定すれば他の機能はすべて seam の上流に層を足す形で安く積み増せるため。

## 帰結

- v0〜v5 のすべての機能追加が「seam の上流に層を足すだけ」で実現でき、核は一度も変わっていない。
- 機能追加のために build の署名や下流を書き換えることは禁止（破るとアンチパターン＝後から全書き直し）。
